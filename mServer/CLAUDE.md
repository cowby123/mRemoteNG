# Claude AI 協作指南 - mServer

## 溝通語言

**重要：本專案與 Claude AI 的所有溝通請使用繁體中文。**

## 專案資訊

- **專案名稱**: mRemoteNG mServer
- **專案類型**: mRemoteNG 設定檔伺服器端
- **開發語言**: Go
- **主要功能**: 提供 API 讓 mRemoteNG 客戶端儲存和讀取設定檔

## 協作規範

### 1. 語言使用
- 所有對話、說明、註解請使用繁體中文
- 程式碼變數名稱、函數名稱保持英文
- 註解和文件說明使用中文

### 2. API 開發規範
- **重要：每次更新或新增 API 時，必須同步更新對應的 Swagger 文件**
- API 變更包括：
  - 新增端點（endpoint）
  - 修改請求/回應格式
  - 修改參數定義
  - 修改錯誤碼
- Swagger 文件必須與實際 API 實作保持一致

### 3. 溝通原則
- 清楚說明需求和問題
- 提供足夠的上下文資訊
- 確認變更前先詢問

### 4. 程式碼規範
- 遵循 Go 語言的慣例和最佳實踐
- 遵循專案現有的程式碼風格
- 變更前先閱讀相關檔案
- 避免過度工程化
- 只進行必要的修改

### 5. Git 操作
- 不要自動提交程式碼，除非明確要求
- 提交訊息使用中文描述
- 遵循專案的分支策略

## 常見任務

- API 開發與維護
- Swagger 文件更新
- 錯誤修復
- 功能開發
- 程式碼重構
- 程式碼審查
- 文件撰寫

## 注意事項

- 避免引入安全漏洞（SQL Injection、XSS、命令注入等）
- 注意 API 的驗證和授權機制
- 確保資料儲存的安全性
- 處理好錯誤回應和狀態碼
- 測試變更是否影響現有功能
- 保持向後相容性

## 開發流程檢查清單

當開發或修改 API 時，請確認：

- [ ] API 功能已正確實作
- [ ] Swagger 文件已更新
- [ ] 錯誤處理已完善
- [ ] 安全性已考慮
- [ ] 測試已通過
